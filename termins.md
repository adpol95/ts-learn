Статическая типизация — проверка соответствия типов данных на этапе компиляции, а не во время выполнения.

Выводимая типизация (Type Inference) — когда TypeScript сам определяет тип переменной на основе присвоенного значения.

Явная типизация — ручное указание типа переменной через двоеточие.

Структурная типизация — система, где совместимость типов определяется их структурой (наличием полей), а не названием.

Супертип (Super type) — более общий, «широкий» тип в иерархии.

Подтип (Sub type) — более узкий, специфичный тип.

Специальные и базовые типы
Any — тип, который отключает проверку типов (позволяет делать что угодно).

Unknown — безопасный аналог any; значение можно использовать только после проверки его типа.

Never — тип для значений, которых никогда не должно быть (например, результат функции-бесконечного цикла).

Void — тип, означающий, что функция не возвращает полезного значения.

Union (Объединение) — тип, позволяющий переменной принимать одно из нескольких значений (через |).

Intersection (Пересечение) — тип, объединяющий несколько структур в одну (через &).

Literal Types (Литеральные типы) — ограничение типа конкретным значением (строкой или числом).

Дженерики и трансформации
Дженерики (Generics) — «типы-параметры», позволяющие создавать переиспользуемые шаблоны кода.

Constraints (Ограничения) — использование extends в дженериках для ограничения списка допустимых типов.

Conditional Types (Условные типы) — логика «если-то» в типах (через тернарный оператор).

Infer — ключевое слово для вывода (захвата) типа внутри условного типа.

Mapped Types (Сопоставимые типы) — создание нового типа путем циклического перебора ключей другого типа.

Utility Types (Утилитарные типы) — готовые инструменты для трансформации типов (Pick, Omit, Partial и др.).

Сужение и операторы
Narrowing (Сужение типов) — процесс уменьшения множества возможных типов до одного конкретного.

Type Guards (Защитники типа) — проверки (typeof, instanceof, is), которые инициируют сужение типа.

Exhaustive Check — прием с использованием never для проверки, что обработаны абсолютно все варианты в switch/if.

Type Assertion (as) — принудительное указание компилятору считать переменную определенным типом.

Satisfies — проверка соответствия объекта типу без потери информации о конкретных значениях (литералах).

Typeof — оператор получения типа из переменной.

Keyof — оператор получения всех ключей объекта в виде Union-строки.

Optional Chaining (?.) — безопасный доступ к свойствам объекта, который может быть null/undefined.

Non-null Assertion (!) — указание компилятору, что значение точно не является пустым.

Прочее
Discriminated Union — Union-тип, где объекты имеют общее поле-метку для их различения.

Перегрузка функций (Function overload) — описание нескольких способов вызова одной функции с разными типами аргументов.

Const Enum — перечисление, которое полностью заменяется значениями в итоговом JS-коде.

Object vs object vs {} — различие между глобальным классом, любым не-примитивом и «почти любым» типом.
